# Polycut

Desktop tool that takes a high-poly 3D model (primarily generated by Meshy.ai) and produces a SketchUp-ready file: low enough poly count to stay performant, with materials an interior designer can make look realistic. The output is later rendered in Enscape. Motivated as a free/cheap alternative to Transmutr.

## Language

**Source model**:
A high-poly 3D mesh produced by Meshy.ai from a text or image prompt. Arrives as a single fused mesh with one baked, UV-mapped texture — no separable pieces. Dense triangle geometry, not clean topology.
_Avoid_: scan, Meshy scan (Meshy generates, it does not scan)

**Part**:
A selectable sub-piece of a model (couch legs, cushions, frame) carrying its own material slot, so the designer can reassign materials per-piece inside SketchUp. The headline value of Polycut. A Source model currently contains zero parts (one fused mesh); creating parts is the hard, post-MVP-1 problem.
_Avoid_: object, group, submesh (overloaded elsewhere)

**Primary user**:
An interior designer who builds client preview scenes in SketchUp and renders them in Enscape. Needs product models (couches, furniture) that aren't available as ready-made SketchUp assets.

**Simplify**:
Reduce a Source model's polygon count enough for SketchUp to stay responsive, while the baked texture survives without smearing or warping and the silhouette still reads correctly. The core MVP-1 operation.
_Avoid_: decimate, reduce, optimize (use "simplify" in UI and discussion)

**Plasticky shine**:
The over-glossy default surface look baked into Meshy exports that reads as fake/plastic once in SketchUp or Enscape. The main material complaint, distinct from missing or stretched textures.
